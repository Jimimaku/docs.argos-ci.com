# Code injection

It can be convenient to inject Javascript into a page to ensure the screenshots' consistency.

For example, the screenshot could be taken when the assets are not fully loaded and induce a flaky test.

In our integrations, we inject the following code before taking each screenshot.

## Inject CSS

Injected style to hide elements and fix unstable rendering.

```js
function injectStyles(document) {
  const css = document.createElement("style");
  css.type = "text/css";
  css.textContent = cssText;
  document.body.appendChild(css);
}
```

## Ensure no busy

Check if there is `[aria-busy="true"]` element on the page.

```js
async function ensureNoBusy() {
  const checkIsVisible = (element) =>
    Boolean(
      element.offsetWidth ||
        element.offsetHeight ||
        element.getClientRects().length
    );

  return [...document.querySelectorAll('[aria-busy="true"]')].every(
    (element) => !checkIsVisible(element)
  );
}
```

## Wait for fonts

Wait for all fonts to be loaded.

```js
function waitForFonts() {
  return document.fonts.status === "loaded";
}
```

## Wait for images

Wait for all images to be loaded.

```js
async function waitForImages() {
  return Promise.all(
    Array.from(document.images)
      .filter((img) => !img.complete)
      .map(
        (img) =>
          new Promise((resolve) => {
            img.onload = img.onerror = resolve;
          })
      )
  );
}
```
