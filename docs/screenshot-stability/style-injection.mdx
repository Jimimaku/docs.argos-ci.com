# Style injection

Browsers have unstable rendering for well-known use cases like scrollbar display, border-radius, etc. Therefore, it can be convenient to hide these components to ensure the screenshots' consistency.

In our integrations, we inject the following styles before taking a screenshot.

```css
/* Hide carets */
* {
  caret-color: transparent !important;
}

/* Hide scrollbars */
::-webkit-scrollbar {
  display: none !important;
}

/* Generic hide */
[data-visual-test="transparent"] {
  color: transparent !important;
  font-family: monospace !important;
  opacity: 0 !important;
}

[data-visual-test="removed"] {
  display: none !important;
}
```

This snippet provides the following attributes to hide unstable HTML nodes:

- `data-visual-test="transparent"` — hide this HTML node from screenshots
- `data-visual-test="removed` — remove this HTML node from screenshots

### Usage

Add the custom attribute to your unstable component:

```html
<button data-visual-test="transparent">Click here</button>
```

Then, [inject the style](/code-injection#inject-styles-in-a-page) into the tested page before taking the screenshot.
