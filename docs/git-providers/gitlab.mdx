# Integrating Argos with GitLab

Enhance your development workflow by integrating Argos with [GitLab](https://gitlab.com). This integration provides visual testing feedback directly within your GitLab pull requests. This guide walks you through the setup process.

## Connecting a GitLab Repository

#### 1. Generate a Personal Access Token in GitLab

Follow [GitLab's documentation](https://docs.gitlab.com/ee/user/profile/personal_access_tokens.html#create-a-personal-access-token) to generate a token. Ensure the permissions <Tag>api</Tag> and <Tag>read_user</Tag> are assigned to this token.

#### 2. Configure the Generated Token in Argos

On Argos, navigate to <Tag>Team Settings</Tag> > <Tag>GitLab</Tag>, enter the generated token and save the changes.

#### 3. Import GitLab Project to Argos

In Argos, navigate to the <Tag>Team projects</Tag> page and select <Tag>Create a new Project</Tag>. Continue by clicking <Tag>Continue with GitLab</Tag> and choose the GitLab repository you intend to integrate.

#### Repository Connection Validation

To verify the successful linkage of your GitLab repository, head to <Tag>Project Settings</Tag> > <Tag>Connect Git Repository</Tag> in Argos. The connected GitLab repository should be displayed.

## Associating an Existing Argos Project with a GitLab Repository

Ensure the [GitLab Personal Access Token has been configured correctly](#connecting-a-gitlab-repository). Then, in Argos, navigate to <Tag>Project Settings</Tag> > <Tag>Connect Git Repository</Tag> and select the desired GitLab repository for association.
